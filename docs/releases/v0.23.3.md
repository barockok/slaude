## v0.23.3 — kb_memoize: batch brain writes

A minor release that replaces the single-page `kb_put_page` brain-write tool with
a batch `kb_memoize`. Recording several related notes previously meant one tool
call + one approval card *per page*; `kb_memoize` writes a whole set under one
approval.

### Changes

- **Brain: `kb_put_page` → `kb_memoize` (batch, up to 20 pages).** The tool now
  takes a `pages` array (1..20), each `{ slug, content, summary }`, and writes
  them all behind a single approval. Each page still goes through `brainCall`, so
  the per-user source is ensured before every write (the v0.23.1 FK fix still
  applies). Empty arrays and over-limit calls are rejected with clear errors.
  `kb_put_page` is **removed**, not aliased — callers use `kb_memoize` with a
  one-element array for single writes. The runtime baseline (KB-first stance) and
  the brain-smoke sim scenario are updated to the new tool.
  (`src/knowledge/mcp-tools.ts`, `src/soul/loader.ts`)

### Internal

- `KB_MEMOIZE_MAX_PAGES = 20` exported from `src/knowledge/mcp-tools.ts`.
- Tests: `tests/brain-mcp-tools.test.ts` — single + multi-page writes under one
  approval, empty-array and >20 rejection, denial writes nothing, and that
  `kb_put_page` is gone. `tests/gateway/core/gateway-branches.test.ts` integration
  call updated to `kb_memoize`. Full suite green (997 pass), `tsc` clean.
