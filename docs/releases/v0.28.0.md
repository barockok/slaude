## v0.28.0 — MCP OAuth loopback: a public redirect_uri, so it works in k8s

v0.27.0 shipped the shared always-on OAuth loopback, but it could only ever
advertise `http://localhost:<port>/callback` as its `redirect_uri`. In-cluster
that's a dead end: the IdP redirects the *user's browser*, which then lands on
*their own* localhost — not the pod. So behind an ingress the loopback never
auto-completed and paste-back stayed the only working path. This release closes
that gap.

### Features

- **MCP OAuth: public redirect_uri for the shared loopback.** `SharedLoopback`
  gains a `publicUrl` option; `register()` now emits `<publicUrl><callbackPath>`
  (trailing slashes normalized) when set, falling back to the bound
  `http://localhost:<port>` otherwise. The listener still binds on a private port
  exactly as before — an ingress fronts `publicUrl` and forwards `/callback` back
  to it, where the signed-`state` demux is unchanged. Token exchange reuses the
  same `flow.redirectUri`, so the authorize and exchange legs stay consistent (an
  OAuth requirement). Wired via new env `SLAUDE_OAUTH_PUBLIC_URL` (trimmed; empty
  → localhost fallback), threaded into the singleton loopback at server boot.

  To switch a deployment from paste-back to loopback auto-complete: set
  `SLAUDE_OAUTH_PUBLIC_URL=https://<host>` **and** unset
  `SLAUDE_OAUTH_REDIRECT_URL` (the latter takes priority in `runConnect` while
  set).
