# v0.18.0 — the brain: gbrain-powered knowledge layer

slaude gets a real memory. This release embeds [gbrain](https://github.com/garrytan/gbrain) (MIT, Bun-native, Garry Tan's agent-brain engine) as a library and rebuilds `slaude_kb` on top of it: scoped per-user knowledge with DB-enforced isolation, approval-gated writes, a knowledge graph, synthesized answers with citations, brain-backed episodic memory, and a nightly maintenance cycle. Two of the four North Star bullets — **Memory** (episodic + semantic) and a piece of **Autonomy** (the agent maintains its own knowledge overnight) — land here.

Design + decision record: [gbrain × slaude finding](../findings/2026-06-10-gbrain-slaude-kb.md). Phases 1-3 of that design ship in this release; Phase 4 (multi-agent shared brain on Postgres) is config-not-code and stays documented-only.

### Features

- **Brain engine (`src/knowledge/brain.ts`).** gbrain's PGLite engine (real Postgres compiled to WASM — pgvector, JSONB, trigram indexes, zero external server) runs in-process under `~/.slaude/brain/`. Boots in ~2s, self-migrates. Every user-facing operation is dispatched with `remote:true` plus a synthetic identity derived from the live Slack context, so **gbrain's own fail-closed SQL scoping enforces isolation** — slaude brokers identity, the database enforces it. Kill switch: `SLAUDE_BRAIN_DISABLED=1` (legacy keyword tools remain).
- **Identity-scoped knowledge (`src/knowledge/scope.ts`).** Slack topology maps onto brain sources: `agent` (the agent's own mind), `shared` (trusted channels), `public`, `user-<id>` (private `/1on1` slices), `kb-<label>` (installed wikis). `/1on1` privacy is now **DB-enforced, not prompt-enforced**: what a user tells the agent in a locked thread is invisible to every other scope — verified by tests (cross-scope search returns empty).
- **slaude_kb v2 tools.** New brain-backed tools alongside the legacy three: `kb_think` (synthesized answer with `[Source: ...]` citations and explicit gaps — an answer, not a chunk list), `kb_search` (hybrid-ready, keyword+graph today), `kb_get_page`, `kb_list_pages`, `kb_graph` (wikilink edges + backlinks), `kb_put_page`, `kb_delete_page`. Keyword search needs zero API keys.
- **Approval-gated writes (`src/knowledge/gated-dispatch.ts`).** gbrain ships audit-only; slaude adds the human gate. Tiers: reads and own-slice writes auto; shared writes raise the familiar Block Kit approval card; deletes always need approval; brain-admin ops (`sources_*`, `purge`, `sync`, schema) are manager-tier with a **hard code-level check** that the approval click came from manager/backup — independent of approver routing config, fails closed when no manager is configured.
- **Brain-backed memory (`src/memory/brain-provider.ts`).** Default memory provider when the brain is enabled (`SLAUDE_MEMORY=sqlite` reverts). Each session's turns persist as timeline entries on a `conversations/<sessionId>` page in the agent's own source — episodic memory lands in the same store the nightly cycle can later consolidate into semantic memory. Failure policy: memory never breaks a turn (prefetch degrades to null, sync to a logged no-op). Cross-session: knowledge written in one thread is searchable from every other thread immediately — visibility is RBAC, freshness is instant.
- **Nightly maintenance (`src/knowledge/brain-cycle.ts`).** slaude's slice of gbrain's dream cycle, in-process (PGLite is single-writer): KB wiki re-sync, link/timeline extraction, orphan report, purge of expired soft-deletes. Runs 03:00 local; `SLAUDE_BRAIN_CYCLE="HH:MM"` or `"off"`.
- **KB wikis indexed, git stays truth.** Installed wikis (`slaude.json`) register as `kb-<label>` sources and sync into the brain at boot + nightly. Git remains the system of record for curated KBs; the brain is the index. Deleting `~/.slaude/brain/` loses nothing curated.
- **Remote embedding, opt-in, provider-generic.** Semantic search activates by env — `EMBEDDING_URL` + `EMBEDDING_API_KEY` (+ `EMBEDDING_MODEL`, `EMBEDDING_DIMENSIONS`) against any OpenAI-compatible `/v1/embeddings` endpoint, mapped onto gbrain's generic passthrough recipe; or gbrain-native provider config (e.g. `ZEROENTROPY_API_KEY`) for non-compatible APIs. Until configured, search is keyword+graph and fully keyless. Decision: embeddings stay remote — no local model daemon in the deploy unit.
- **Soul baseline: KB-first leads with the brain.** The runtime baseline now steers `kb_think`/`kb_search` before acting and `kb_put_page` for durable knowledge.

### Fixes

- **Boot race into duplicate sources (`fix(kb)`, found by sim smoke).** Gateway boot and the memory provider both seeded brain sources concurrently; the list-then-add race violated `sources_pkey` and broke first-turn memory prefetch. `ensureSources` is now single-flight and duplicate-tolerant, with the guard reset on `closeBrain`.

### Internal

- gbrain pinned as a git dependency (`github:garrytan/gbrain#03ffc6e` — the npm package named `gbrain` is unrelated). tsc never descends into gbrain's sources (erased dynamic import specifiers); slaude's strict typecheck stays clean.
- 71 new tests across scope/gating/engine/tools/memory/sync/cycle, including real-PGLite integration tests (~2s each). Suite: 792 pass.
- Real-agent sim scenario `src/gateway/sim/scenarios-real/brain-smoke.yaml` (write → approval card → approve → scoped search round-trip); verified passing against the live agent. Kept outside the stub CI glob.
- Implementation plan: `docs/superpowers/plans/2026-06-10-gbrain-slaude-kb-phase1.md`.
- Known gap (documented): wikilinks inside fs-synced wiki sources don't produce graph edges yet (gbrain extraction resolution — upstream investigation); agent `kb_put_page` writes reconcile edges inline, wiki search unaffected.
