## v0.22.0 — KB-first means it, and a quieter resume

A small but pointed release: the knowledge-base directive moves from polite
suggestion to runtime contract, and a scary-but-harmless warning stops leaking
into Slack when an old session migrates onto resumable-sessions. Plus the design
for actually *enforcing* KB-first, captured for a later build.

### Features

- **Soul: KB-first is now mandatory, not advisory.** The `RUNTIME_BASELINE`
  KB-first stance was a self-judged conditional ("when you lack context… query
  the KB"), so a confident-but-wrong agent could answer straight from its
  training and the runtime never noticed. It's rewritten in the imperative
  register the approval gate uses: before any substantive answer or non-trivial
  action — and **always** before a mutation — the agent MUST query the KB first
  (default `kb_search`, or `kb_think` for a cited synthesis), must not answer
  from its own weights when brain tools are mounted ("your training is not this
  team's source of truth, the KB is"), and must state gaps explicitly instead of
  guessing. Pure chitchat / acknowledgement ("Hi", "thanks 👍") stays exempt.
  Lives in the non-overridable baseline, injected every turn.

### Fixes

- **Agent: the resume-miss warning no longer leaks to Slack.** A session created
  before resumable-sessions has `claude_started=1` but no CLI transcript under
  slaude's id, so its first boot resume-misses. That self-heals — `#startSession`
  clears the flag and reboots seeding `--session-id`, so the new claude
  conversation lands under slaude's id and later resumes hit. But the miss also
  arrived as a `result(is_error)`, which `#fanout` emitted unconditionally — so
  the user saw a `:warning: error: No conversation found with session ID: …`
  for an expected, already-handled condition. `#fanout` now suppresses **only**
  the resume-miss (shared `RESUME_MISS_RE` with the retry path), logging a
  `[resume-attempt]` line server-side instead; every other turn error still
  surfaces. One-time, now-invisible migration for pre-feature sessions.

### Internal

- **Docs: KB-first enforcement design.** Follow-up to the wording hardening above
  — how to add a Stop-hook *forcing function* without nagging on "Hi". Tiered
  design (hard-enforce only on substantive-work-without-`kb_*`, trust prose for
  pure replies), shadow/metric-mode rollout, open questions. Designed, not
  implemented. See
  [docs/findings/2026-06-13-kb-first-enforcement.md](../findings/2026-06-13-kb-first-enforcement.md).
