# v0.20.0 — dream the backlog

### Features

- **`bun run brain-backfill` — import historical sessions into the brain.** Deploys carry weeks of pre-brain conversations in sqlite `memory_turns`; the brain never saw them. The backfill imports every session as a `conversations/<id>` page in the agent source — full transcript in the page body (chunk-indexed: keyword/vector searchable) plus timeline entries matching what `BrainMemoryProvider` writes for live turns. Idempotent: deterministic page content + gbrain's timeline dedup make re-runs no-ops. Run once in the pod terminal (`--no-embed` to skip the embed pass). (`src/knowledge/brain-backfill.ts`, `src/cli/brain-backfill.ts`)
- **Embed-stale-chunks sweep, post-backfill and nightly.** Chunks created by `kb_put_page`/memory writes were never embedded (sync only embeds fs-sourced pages). The sweep (`runEmbedCore({stale:true})`) runs after backfill and as a new nightly maintenance phase whenever the embedding gateway is active — closing the v0.18.0 known gap. (`src/knowledge/brain-cycle.ts`)
