## v0.32.0 — MCP stream recovery, per-channel cost metrics, cron credential scoping

This release hardens two long-standing operational pain points (MCP transport drops mid-session, cron jobs inheriting wrong credentials) and adds the per-channel token/model metrics needed for downstream cost attribution.

### Features

- **Agent: stream_closed auto-recovery with circuit breaker.** When an external MCP server disconnects mid-turn, the SDK's shared transport pool tears down, causing all in-process slaude_* tool calls to fail with "Stream closed". Previously this silently orphaned the turn. Now `AgentManager` detects the error in the `tool_use_result`, defers action to turn-end, and automatically reloads the session — injecting a synthetic "The MCP stream was restored. Please continue where you left off." prompt so the resumed session picks up without human input. A per-session circuit breaker (max 2 auto-reloads) prevents infinite reload loops; on the third consecutive failure the circuit opens and posts a visible error via the Bolt WebClient (bypassing the broken MCP surface). See `src/agent/manager.ts`.

- **Metrics: per-channel token and model labels.** Token counters (`slaude_tokens_total`) now carry `channel_id` and `model` labels on every increment, enabling downstream Prometheus queries to break cost by channel and by model. Previously all increments were label-less, making channel-level cost attribution impossible. See [findings doc](../findings/2026-07-08-per-channel-metrics.md).

- **Cron: private-MCP credential scoping by initiator.** Cron jobs created inside a `/1on1` session previously inherited the wrong MCP credentials at runtime — the live lock's config dir was used instead of the initiator's. `resolveEffectiveIdentity()` now unifies both code paths (config-dir selection and MCP credential scoping) through a single lookup keyed to the cron initiator. See [findings doc](../findings/2026-07-03-cron-private-mcp-scoping-drift.md) and PR #66.

### Fixes

- **Metrics: model label derived from `modelUsage` keys.** The `SDKResultMessage` type has no `.model` field; the prior code always resolved `undefined`. Fix derives the model name from `Object.keys(modelUsage)[0]` which reflects the actual model used. (#67 NIT)

- **Metrics: `row` reference error in `#fanout`.** The per-channel label fix accidentally used `row.slack_channel_id` directly inside `#fanout`, where `row` is not in scope. Corrected to `Sessions.findById(sessionId)?.slack_channel_id`. (`3d8c77a`)

### Internal

- **Tests: stream_closed recovery coverage.** Four new cases in `manager-lifecycle.test.ts`: happy-path auto-reload + auto-continue, circuit breaker (opens at attempt 3), array-format content block detection, and `abort()` mid-turn clearing pending state. (`7854b87`)

- **Tests: cron private-MCP scoping.** New `cron-private-mcp-scoping.test.ts` covering the identity resolution paths for both live-lock and initiator-based credential selection. (PR #66)
