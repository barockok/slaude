## v0.8.11 — KB-first soul + other-bot visibility

Behavioral release. Tightens how slaude reaches for context and widens what it can hear in shared Slack threads. No schema or config breaks.

### Features

- **Soul: KB-first stance baked into runtime baseline.** When the agent lacks context — unfamiliar domain, ambiguous request, or before any non-trivial mutation — it must query the knowledge base before acting. Reduces wrong-assumption risk by surfacing operator-curated precedent first. Lives in `src/soul/loader.ts`'s `<runtime-baseline>`, non-overridable by SOUL.md.
- **Slack: other bot users now visible in threads.** Previously every event with `bot_id` or `subtype=bot_message` was silently dropped, blinding slaude to CI alerts, summarizer bots, and any other bot in shared threads. Now self bot_id is captured at startup from `auth.test`, and only self-echoes are dropped (metric label `self_bot`). Other bots flow through the normal blocklist → channel-trust → engagement gates. In DM/unlisted channels they still get filtered at the whitelist gate (manager-only).

### Internal

- New `docs/releases/<tag>.md` convention for hand-written release notes. Release workflow prefers this file over the auto-generated git-log dump when present. Convention now in `CLAUDE.md` working rules.
