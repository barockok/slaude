# v0.20.1 — resume fixed + baseline indexing awareness

### Fixes

- **Idle-reopen resume actually works now.** slaude generated one session uuid (sqlite) while the claude CLI generated another for the conversation; `resume` asked for slaude's id, which the CLI had never seen — a 100% miss masked by the fresh-boot fallback. Every thread waking from idle logged `No conversation found with session ID`, cold-started, and silently lost SDK-side context (tool results, compaction state). First boot now seeds the CLI with slaude's id (`--session-id` via the SDK's `extraArgs`); resume finds the transcript. Existing mismatched sessions self-migrate on their first reopen via the not-found recovery; a mirror recovery covers the seeded-id-already-has-a-transcript edge. (#19)

### Features

- **Baseline documents the brain's auto-indexing.** Agents now know KB wikis index at boot + nightly (so `kb_search`/`kb_think` already cover wiki content), that mid-day wiki changes lag until the next index pass (verify with `Read`/`Grep` when freshness matters), and the precise memory contract: turns persist + recent turns auto-recall, but only explicit `kb_put_page` writes are searchable later. (#18)
