# v0.19.2 — fix: brain boot after unclean pod shutdown

Second patch for the maria UAT rollout ([v0.19.1](v0.19.1.md)).

### Fixes

- **`GBrain: Timed out waiting for PGLite lock` on pod restart.** gbrain guards its PGLite data dir with a pid-file lock and checks staleness via `kill(pid, 0)`. In Kubernetes the previous container's recorded PID routinely maps to a live process in the new container's PID namespace, so a lock left by an uncleanly-stopped pod never looks stale — boot blocks 60s, then the brain bootstrap fails. slaude now clears any leftover lock before connect: by deploy contract exactly one process owns a brain home (one container = one persona, `Recreate` strategy), so a lock present at fresh-process boot is stale by construction. Opt out with `SLAUDE_BRAIN_TAKEOVER=0` if you deliberately share a brain home across processes (don't — PGLite is single-writer). (`src/knowledge/brain.ts`)
