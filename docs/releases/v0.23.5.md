## v0.23.5 — /help as an aligned code block

A patch release improving the `/help` output. The command list rendered as inline
markdown (backtick-wrapped usage + em-dash + summary), which wrapped unevenly. It
now renders as a monospace code block with summaries aligned to a single column.

### Changes

- **`/help` renders an aligned code block.** `helpText()` emits the command table
  inside a code fence, each line `usage.padEnd(gutter) + summary` where
  `gutter = longest usage + 2`. Slack renders fences monospace, so the
  space-padding lines summaries up cleanly. Usages are no longer backtick-wrapped
  (backticks show literally inside a fence). Tabs are intentionally avoided — they
  render at inconsistent widths across Slack clients. The format survives
  `mdToMrkdwn` (fenced blocks are carved and restored verbatim). The
  tool-permission modes list is unchanged.
  (`src/gateway/slack/commands.ts`)

### Internal

- Test: `tests/commands.test.ts` — `/help` contains a fence and every command
  line is padded to the shared gutter. Full suite green (998 pass), `tsc` clean.
