## v0.29.0 — Session self-awareness, agent-driven modes, and quieter Slack

A batch focused on the agent *knowing* and *steering* its own engagement state, plus
two Slack-surface fixes. The model can now see when it's in a private `/1on1`, learns
about out-of-band state changes (connect/disconnect, model, mode, soul, cron) as they
happen, and can start/release `/1on1` and a new mention-only mode itself.

### Features

- **Session awareness in the system prompt + timeline (#54).** The model had no idea
  what mode it was in. Now: a `<session-mode>` block names the locked user when a
  thread is `/1on1`-locked (recomputed on the reboot `/1on1` triggers — the system
  prompt is sent per-request, not frozen in the resumed transcript). And gate commands
  that run *out-of-band of the model* now drop a one-time marker into the next turn's
  context so the agent stays aware: `/mcp connect`/`disconnect`, `/model`, `/mode`,
  `/soul`, `/cron-add`/`-remove`. Delivered exactly once via a `UserPromptSubmit` hook;
  never spawns a model run.
- **`slaude_1on1` agent tool (#57).** `/1on1` was human-only. The agent can now start
  or release a 1on1 itself (`mcp__slaude_surface__set_one_on_one`) when a user asks to
  go private — same engine as the slash command, locked to the current speaker, no
  gating. Mounted in the `slaude_surface` family.
- **Mention-only mode (#59).** A new per-thread posture: `/mention-only [off]` (or the
  agent-facing `mcp__slaude_surface__set_mention_only`) makes the agent reply *only* to
  messages that @-mention it and ignore plain follow-ups — like disengage, but
  persistent and conditional on a mention. Plain messages are recorded-suppressed if a
  session exists (context for when next tagged), else dropped. No gating.

### Fixes

- **`/unignore*` deadlock (#56).** The ignore gate dropped messages *before* slash
  parsing, so a thread-ignore swallowed the very `/unignore-thread` meant to lift it —
  the thread was stuck ignored forever. `/unignore*` now bypasses the ignore gate (the
  blocklist + channel-mode + 1on1 gates after it still apply). The old test masked this
  by not thread-pinning its sends.
- **Compact Slack links (#58).** Agent-emitted bare URLs rendered as full raw URLs and
  ate whole lines. `mdToMrkdwn` now rewrites a standalone bare `http(s)` URL to a
  labeled `<url|host>` link — compact visible text, full URL preserved as the target.
  Markdown links, mrkdwn links, and autolinks are left untouched.

### Internal / Deprecations

- **`/ingest` soft-deprecated (#55).** Superseded by brain memoize (gbrain captures
  knowledge automatically). The command now replies a deprecation notice and no longer
  runs; dropped from `/help`; `knowledge/ingest.ts` + `db/ingest-jobs.ts` marked
  `@deprecated`. Code + `kb_ingest_jobs` table kept — full removal is a follow-up.
- New `mention_only_threads` table; `set_one_on_one`/`set_mention_only` folded into the
  `slaude_surface` tool family (no standalone servers). Full suite 1147 pass.
