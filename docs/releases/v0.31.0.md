## v0.31.0 — live todo tracker + cron OAuth fix for /1on1

Two changes in this release: todo tasks are now visible in Slack as the agent
works through them, and a cron bug that caused OAuth-authenticated MCP tokens
to be silently dropped for jobs created inside a /1on1 is fixed.

### Features

- **Gateway: live todo tracker in the Slack thread (#65).** The gateway's
  tool-call interceptor catches every `TodoWrite` event and renders a compact
  markdown task list in-thread with per-status icons — ✅ completed, ▶ **bold**
  in-progress, ○ pending. The first write posts the block; subsequent writes
  edit it in-place so the thread stays clean. When the turn ends with every
  item completed the heading is stamped `Tasks ✅`. Tracker state is cleared at
  the start of each new user turn so each request gets a fresh block posted
  after the user's message. (`src/gateway/core/gateway.ts`)

### Fixes

- **Cron: reuse initiator OAuth for jobs created in a /1on1 (#64).** Jobs
  created inside a /1on1 persist `slack_thread_ts=null`, so the scheduler keyed
  their run on a synthetic `cron:<id>` thread that carried no 1on1 lock.
  `#startSession`'s lock lookup missed, and the child booted under the agent's
  OAuth config dir instead of the initiator's — OAuth-authenticated MCP tokens
  weren't reused. Fix: capture the active 1on1 lock owner at `/cron-add` time
  onto a new nullable `cron_jobs.oauth_user` column (null = created outside a
  1on1, unchanged behavior). The scheduler hands that identity to `AgentManager`
  via `setCronOAuthUser` before the run, and `#startSession` prefers it over the
  thread-lock lookup when resolving `CLAUDE_CONFIG_DIR`. DB schema bumped (new
  column, nullable, no migration required for existing rows). (`src/agent/manager.ts`,
  `src/db/cron-jobs.ts`, `src/db/schema.ts`, `src/gateway/slack/cron-scheduler.ts`)

### Internal

- `.playwright/` added to `.gitignore` (playwright-cli scratch dir).
