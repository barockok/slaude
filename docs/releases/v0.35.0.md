## v0.35.0 — Per-agent brain slices: private minds, deliberate sharing

Each agent now has its own private knowledge slice (`agent-<id>`) that auto-passes the write gate. The long-standing friction — approval cards on every in-conversation `kb_memoize` — is gone. The model: classify the write *destination*, not the unknowable intent behind it.

### Features

- **Brain: per-agent private slices (`agent-<id>`) with auto-pass gate.** The runtime baseline tells agents to proactively record durable knowledge, but the write gate carded the human on every in-channel `kb_memoize`. Root cause: the only auto-pass paths were background/cron turns (`userId === null`) and `/1on1` own-user slices. Normal Slack turns always carry `ctx.userId`, so agent-initiated writes were attributed to the human and required approval. Fix: agents get their own `agent-<id>` source, which auto-passes exactly as a user's own slice does inside `/1on1`.

- **Brain: `kb_memoize` `target` param (`"mine"` | `"shared"`).** Default is `"mine"` — routes to the caller's own slice and auto-passes. Passing `target:"shared"` explicitly routes to the shared team KB and raises an approval card. The authorization input is the *destination*, not self-attested intent, so there's nothing for the model to game. Writing to shared is the deliberate act worth a card.

- **Brain: agent identity resolution (`agent-identity.ts`).** Agent id resolves once at boot from three sources in priority order: `SLAUDE_AGENT_ID` env (recommended for multi-agent deploys), `auth.test` on the posting token (bot user id or post-as-user id), and `"default"` as a single-agent/test fallback. The resolved id is cached process-wide; early memory writes await resolution so nothing splits across `agent-default` and `agent-<real-id>`.

- **Brain: legacy `agent` source kept read-only for continuity.** Rather than renaming the existing single-agent `"agent"` source destructively (security-sensitive code + data migration risk), the legacy source stays in each agent's read union. Existing memory stays visible while new writes accrete in `agent-<id>`. A consolidation rename is a later opt-in CLI op.

- **Gateway: boot resolution + `brainGateFor` per-agent wiring.** `gateway.ts` resolves agent identity at boot and threads the agent id into the brain gate, so every session carries the right slice scope from first turn.

### Internal

- Tests: `agent-identity.test.ts` covers all three resolution paths, caching, and reset. `brain-gate.test.ts` + `brain-mcp-tools.test.ts` cover the agent-slice auto-pass, `target` param routing, and isolation between agents.
- CI: tolerate bun 1.3.11's non-zero quirk exit (codes 99/100) in the test runner step. Drop the version-flaky `statement` sub-threshold from the coverage gate.
