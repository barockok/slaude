## v0.27.0 — MCP OAuth connect: one engine, two front doors, Slack-safe

This release reworks how the agent connects OAuth-backed MCP servers. The old
`/mcp connect` flow had Slack hard-coded into it and a fresh per-flow loopback
listener; the markdown→mrkdwn converter quietly corrupted the authorize URL. All
three are fixed, and the agent can now connect a service from natural language —
without ever weakening the security model: the authorize URL never passes through
the LLM, no paste is needed, and every flow tears down cleanly.

See `docs/findings/2026-06-19-shared-oauth-loopback-surface-connect.md` for the
full investigation (including why the runtime *synthesizes* the OAuth tools the
server never ships, and why exact-match `redirect_uri` + open dynamic registration
makes a single shared callback port work).

### Features

- **MCP OAuth: shared always-on loopback, demuxed by signed state.** A single
  callback listener (fixed port, default 3118) serves every session's OAuth
  callback, routing each by an HMAC-signed `state` that carries the session id
  (RFC 6749's client-set, echoed-back slot). Many sessions can authorize
  concurrently behind one pre-mapped port, instead of a fresh ephemeral listener
  per connect. Opt-in via `SLAUDE_OAUTH_SHARED_LOOPBACK` (+ `_PORT`,
  `SLAUDE_OAUTH_STATE_SECRET`); the per-flow path remains the default. New modules
  in `src/agent/mcp-oauth/`: `state.ts`, `shared-loopback.ts`, `shared-client.ts`,
  `token-exchange.ts`.
- **MCP OAuth: natural-language connect.** A new agent-facing tool
  `mcp__slaude_connect__connect_mcp(server)` lets the agent connect a service when
  a user asks ("connect workbench"). It routes into the *same* deterministic
  `connectServer` engine as `/mcp connect` — scope-gated (you connect your own
  services in a `/1on1`; the agent's shared identity is manager-only),
  fire-and-forget, returning only a status line. The authorize URL is posted
  out-of-band by the gateway and the loopback captures the result, so the model
  never sees the link and the user never pastes anything.
- **Surface-aware connect.** `/mcp connect` (loopback, paste-back, and button)
  now posts through the session's Surface instead of a hard-coded Slack client —
  channel-agnostic (Slack, sim, future surfaces), and it runs even outside a live
  turn.

### Fixes

- **Slack mrkdwn: emphasis flanking.** The converter did a pure `**`→`*` swap that
  Slack's renderer rejects. `***x***` → `_*x*_` (Slack nests bold+italic, has no
  triple-star), `** x **` → `*x*` (inner padding trimmed — Slack won't emphasize
  with a space hugging the marker). Intra-word bold (`word**b**word`) is a
  Slack-renderer limit, left as-is.
- **Slack mrkdwn: URLs corrupted by emphasis rules.** Base64url query params
  (OAuth `code_challenge`/`state`) contain `__`, so a posted authorize URL got
  rewritten mid-string. `mdToMrkdwn` now carves `<…>` autolinks and bare
  `http(s)://` tokens (protect + restore, like code spans) before the link rule —
  fixing any agent-emitted URL, not just OAuth.

### Security

- **Auth-URL redaction on teardown.** Once a connect flow settles — success,
  failure, *or* loopback timeout — the authorize-URL message is edited in place to
  strip the live link (redact, not delete), so no stale clickable secret lingers
  in the thread.
- **State HMAC verified before routing.** The shared loopback verifies the
  inbound `state`'s HMAC *before* registry lookup (wired to `verifyState`), so a
  tampered/forged state is rejected before it can match a pending flow — closing a
  latent forgery surface flagged in security review.
- **Synthesized OAuth tools denied.** The permission gate hard-denies
  `mcp__*__authenticate` / `__complete_authentication` (the runtime/SDK-synthesized
  tools that use an uncontrolled redirect and would launder the URL through the
  model) — before the approval prompt, so they can't run and aren't even offered.
  Every connect is forced through our engine.

### Internal

- New test suites: `tests/mcp-oauth/` (state, shared-loopback, shared-client),
  `tests/connect-mcp.test.ts`, plus expanded gateway/permission-gate/format coverage.
- Dropped an unreachable decode `try/catch` in `verifyState` (base64url decode is
  lenient and the MAC verifies first) → `state.ts` 100%.
