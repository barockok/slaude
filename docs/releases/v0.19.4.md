# v0.19.4 — fix: embedding gateway bootstrap

Fourth patch in the maria UAT rollout series ([v0.19.3](v0.19.3.md)).

### Fixes

- **Server died with `Embedding gateway is not configured` once embeddings were enabled.** gbrain's embedding gateway is a process singleton initialized by `configureGateway()` — writing `embedding_model` to `config.json` (what `applyEmbeddingEnv` does) isn't enough on the library-embed path, and a sync embed step hitting the unconfigured gateway exits the process. Brain boot now calls `configureGateway(buildGatewayConfig(loadConfig()))` when an embedding model is configured **and** its provider key env (e.g. `ZEROENTROPY_API_KEY`) is actually present; KB sync gates its embed step on that runtime readiness flag (`embeddingActive()`), not the config file. A configured model with a missing key now degrades to keyword-only search with a boot warning instead of killing the server. (`src/knowledge/brain.ts`, `src/knowledge/brain-sync.ts`)
