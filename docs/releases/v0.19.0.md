# v0.19.0 — subscription-friendly kb_think + native embedding providers

Fast follow to [v0.18.0](v0.18.0.md), unblocking the brain on deploys that authenticate with a Claude subscription token (no raw `ANTHROPIC_API_KEY` in the pod) — which is exactly how the maria deployment runs.

### Features

- **`kb_think` synthesis routed through the claude-agent-sdk.** gbrain's own think path calls the raw Anthropic SDK, which only accepts `ANTHROPIC_API_KEY` — and putting that key in the pod env would silently flip the *main agent loop* off subscription OAuth too (explicit key beats token). Instead slaude injects a `ThinkLLMClient` shim into gbrain's `runThink`: gather (pages + takes + graph + trajectory, fully source-scoped) stays gbrain's; the synthesis call becomes a one-shot, tool-less SDK query that reuses whatever auth the deploy already has (`CLAUDE_CODE_OAUTH_TOKEN` or `ANTHROPIC_*` gateway env). One auth decision, made once, honored everywhere. gbrain's resolved model id is intentionally ignored — the SDK session default rules. (`src/knowledge/brain-think.ts`)
- **Provider-qualified embedding models without a URL.** `EMBEDDING_MODEL=zeroentropyai:zembed-1` (any `provider:model` id) now activates embeddings directly through gbrain's native recipe — no `EMBEDDING_URL` needed; the recipe resolves its own endpoint from its provider key env (e.g. `ZEROENTROPY_API_KEY`). Default dims follow the provider (2560 for ZeroEntropy) unless `EMBEDDING_DIMENSIONS` overrides. The OpenAI-compatible `EMBEDDING_URL` path from v0.18.0 is unchanged.
