## v0.10.0 — Gate Commands as Session Tools

Agent can now manage cron jobs, trigger KB ingest, and unignore threads directly via MCP tools — no slash commands needed. Authorization still enforced against SOUL.md manager/approver lists.

### Features

- **Slack: expose gate commands as MCP session tools.** Five new tools let the agent self-manage where reasonable:
  - `list_cron_jobs` — read-only audit of active scheduled jobs. Agent can tell users what's running.
  - `add_cron_job` — schedule recurring prompts (e.g. "daily summary at 9am UTC"). Manager/approver auth required.
  - `remove_cron_job` — cancel a scheduled job by ID. Manager/approver auth required.
  - `trigger_ingest` — KB raw→wiki sync on demand. Manager/approver auth required.
  - `unignore_thread` — resume a previously ignored thread. No auth; agent self-regulation.
- **Slack: userId plumbed into SlackContext.** Adapter passes the current turn's Slack user ID through to MCP tools, enabling authorization checks that mirror gate-level logic. Used by all admin-facing tools.

### Internal

- `abort` and `mode` remain gate-only — human must control these, not agent.
- Tool descriptions embed use-case guidance so the agent knows when to invoke each.
