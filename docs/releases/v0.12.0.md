## v0.12.0 — Vercel-style skill sources + cron event fix

This release adds Vercel `npx skills` compatible source references to `slaude.json`, so skills can be declared as `owner/repo[/path][@ref]` instead of full `git`+`ref`+`path` tuples. Also fixes a cron scheduler bug where job completion was never recorded because the scheduler listened to the wrong agent event names.

### Features

- **Skills: Vercel-style `source` field.** `slaude.json` skill entries now accept `"source": "owner/repo[/path][@ref]"` instead of verbose `git`+`ref`+`path`. Resolved at install and sync time to `github:owner/repo` with inferred `ref` (default `main`) and optional `path`. `src/config/manifest-schema.ts`
- **Skills: worktree detection.** Skills inside `.claude/skills/` (Claude Code worktree layout) are discovered alongside `~/.slaude/skills/`. `src/skills/loader.ts`
- **Install & sync: source normalization.** Both `install-deps` and `syncManifest` normalize `source` → `git`+`ref`+`path` before clone or lockfile generation. `src/cli/install.ts`, `src/skills/sync-manifest.ts`

### Fixes

- **Cron: listen to `"event"` payloads, not raw `"done"`/`"error"`.** AgentManager emits `{ type: "done", sessionId }` via the `"event"` event. CronScheduler was registering `on("done")` and `on("error")` which never fired, so jobs stayed in `#running` forever and completion was never recorded. Now listens to `"event"` and checks `e.type`. `src/gateway/slack/cron-scheduler.ts`

### Internal

- Tests for `resolveSkillSource`, `resolveSkillSlug`, schema validation, and sync-manifest source normalization. `tests/manifest-schema.test.ts`, `tests/sync-manifest.test.ts`
- Cron scheduler tests updated to emit `"event"` payloads matching real AgentManager behavior. `tests/cron-scheduler.test.ts`
