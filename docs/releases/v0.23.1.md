## v0.23.1 — brain memoize: durable writes in /1on1, honest recall

A patch release that repairs the *remember / think / explore → brain* loop. Three
UAT cases (agent "Maria" at Amartha) surfaced the same symptom — "you don't
remember what I told you" — hiding **two distinct defects**. Forensics on the
session JSONL plus a live PGLite snapshot isolated both. Full writeup:
[2026-06-14 brain-memoize-failure](../findings/2026-06-14-brain-memoize-failure.md).

### Fixes

- **Brain: `/1on1` memoize no longer FK-fails (write-never-lands).** Every
  `kb_put_page` inside a `/1on1`-locked thread silently failed on
  `pages_source_id_fkey`. The lock scopes writes to a per-user source
  (`user-<id>`), but that source is only ever *resolved* at write time —
  `ensureSources()` bootstraps `agent/shared/public/kb-*` and never the user
  slices. Confirmed against the live UAT brain: the `sources` table held no
  `user-*` rows; the one *unlocked* write (to `shared`) succeeded, isolating the
  lock as the cause. `brainCall` now ensures `scope.sourceId` exists before any
  scope-write op (`isScopeWriteOp`); `ensureSource` idempotently registers it
  (swallows `source_id_taken`/duplicate), cached per id. The bug bit exactly
  where durable memory matters most — a focused 1:1 "remember this."
  (`src/knowledge/brain.ts`, `src/knowledge/gated-dispatch.ts`)

- **Brain: `kb_think` stops false "not captured" (write-lands-recall-misses).** A
  present, indexed, non-stale page was reported missing because `kb_think`'s
  synthesis gathered 40 candidate pages and the target lost the ranking race on a
  verbose query — while a tight-keyword `kb_search` found it at rank 1. `kb_think`
  now falls back to keyword `kb_search` when synthesis returns zero citations,
  attaching `search_fallback` hits so known knowledge is never declared absent.
  (`src/knowledge/mcp-tools.ts`)

### Internal

- Regression tests: `tests/brain.test.ts` (put_page auto-ensures an
  un-bootstrapped user source, real PGLite) and `tests/brain-mcp-tools.test.ts`
  (kb_think fallback on zero citations; no fallback when citations exist; empty
  and erroring fallback paths). Full suite green, `tsc` clean, codecov/patch pass.
- `scripts/debug-brain-fk.sh` — read-only in-pod PGLite snapshot-and-query tool
  used to confirm the missing `user-*` source on the live UAT brain. Safe by
  design: copies the single-writer data dir before querying, never touches live.
- Deferred (lower-leverage, noted in the finding): actionable FK→agent error
  mapping, a failed-intent brain-write stop-guard, and deeper `kb_think`
  candidate-set ranking.
