## v0.15.0 — per-user OAuth: connect your own MCP servers in /1on1

This release makes `/1on1` the home for personal credentials. A locked thread already runs as its initiator (per-initiator `CLAUDE_CONFIG_DIR`); now the initiator can also stand up their own OAuth-backed HTTP MCP servers from inside the thread with `/mcp connect`, and slaude clears the agent's own secrets for whitelisted private services so the session truly acts *as the user*. The borrow-on-behalf-of "connect broker" — designed before the per-user lock existed — is removed: the lock makes it obsolete.

Alongside the credential work, the simulation harness graduated to a real interactive REPL on OpenTUI (React), decoupled from Slack by a new Surface abstraction, and cron jobs learned to post to a channel root instead of only a thread.

### Features

- **/mcp OAuth connect in /1on1.** The lock initiator runs `/mcp` to list connectable HTTP MCP servers and `/mcp connect <server>` to authorize one. slaude drives the full OAuth handshake itself (AS discovery → dynamic client registration → PKCE+state → token exchange) and writes the result into the CLI's *native* `mcpOAuth` store in the initiator's config dir, so claude-code owns refresh/reconnect/lifecycle — slaude does only the one-time grant the SDK exposes no hook for. Gated three ways: slash, button, and pasted callback are all initiator-only. See [finding](../findings/2026-06-09-mcp-oauth-connect-1on1.md).
- **Two callback modes — loopback and paste-back.** Loopback (default, local/same-host) binds an ephemeral listener. Paste-back (`SLAUDE_OAUTH_REDIRECT_URL`, for k8s/remote where no runtime port is reachable) posts the authorize URL, parks a `state`-bound pending flow keyed `channel:thread:user`, and finishes when the initiator pastes the callback URL/code back into the locked thread — intercepted before the model ever sees it.
- **Boot canary disables /mcp on store-format drift.** The CLI's `mcpOAuth` key derivation (`a2A`) is pinned by a golden canary asserted at startup; if a CLI upgrade changes the format, `/mcp` connect self-disables instead of writing tokens the CLI can't read.
- **Private services in /1on1.** A `privateServices` whitelist clears the *agent's* own credentials for those MCP servers in a locked thread, so the server runs purely as the initiator. The session reloads on lock/release to re-evaluate. See [finding](../findings/2026-06-08-private-services-1on1.md).
- **Per-initiator OAuth isolation.** A locked session runs with `CLAUDE_CONFIG_DIR` set to the initiator's isolated home (`$SLAUDE_HOME/oauth/<userId>`), so OAuth MCP servers connect as that user, not the shared agent. See [finding](../findings/2026-06-08-oauth-config-dir-1on1.md).
- **cron-on-channel.** `/cron-add` accepts an optional `[channel|thread]` target; channel jobs broadcast to the channel root instead of a thread. See [finding](../findings/2026-06-08-cron-on-channel.md).
- **Interactive sim REPL on OpenTUI (React).** The simulation gateway now renders a claude-code-grade terminal REPL — live status, gate box, group activity, pinned bordered input, inspection commands (`/budget` `/memory` `/sessions`), and a `--real` mode that drives the live agent. Scenarios compose as layer × role × behavior. See [findings](../findings/2026-06-05-sim-tui-opentui.md).
- **Surface abstraction.** Agent interaction is decoupled from Slack behind a `Surface` port (`SlackSurface` + capability-gated MCP), so the sim drives the same canonical reply path as production. See [finding](../findings/2026-06-03-surface-abstraction.md).

### Fixes

- **Keychain defeats CLAUDE_CONFIG_DIR on macOS.** The CLI's store reader is keychain-primary on darwin, shadowing the `.credentials.json` write — so connect→connected round-trips only on Linux/container. slaude warns instead of silently no-op'ing; the verify harness SKIPs on darwin.
- **Keep the initiator's `.credentials.json`.** `ensureInitiatorConfigDir` no longer scrubs that file — the `/mcp connect` flow writes the initiator's own token store there, and the scrub ran on every locked-session boot.
- **Sweep abandoned paste flows.** Each pending paste entry closes over live client creds + the PKCE verifier; expired entries are now evicted on a periodic sweep, not lazily. Bare-code match raised to ≥20 chars so ordinary one-word replies aren't consumed as a code mid-connect.
- **Scheduler boot TDZ crash.** Start the cron scheduler *after* the routes map is declared. See [finding](../findings/2026-06-08-cron-on-channel.md).
- **Sim real-agent slash commands** no longer wait on the 180s turn timeout.

### Internal

- **Removed the connect-broker / `slaude_connect` family.** Deleted the broker module, connect tools, connections/grants/audit tables, the `SLAUDE_ENABLE_CONNECT_BROKER` flag, the encryption-key apparatus, and the grant/borrow extensions to `ApprovalGate`. The per-initiator lock supersedes it.
- New `src/agent/mcp-oauth/` module (pkce, discovery, register, loopback, callback, client, store) with the SDK-utilization boundary documented in the spec.
- `verify-mcp-oauth.sh` + reshaped `verify-1on1.sh`; docker harness verifies OAuth isolation on Linux.
- 734 tests pass, tsc clean.
