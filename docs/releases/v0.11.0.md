## v0.11.0 — Cron thread sessions + ignore gate hardening

This release implements every fix from the [2026-05-25 Cron & Ignore System Audit](docs/findings/2026-05-25-cron-ignore-audit.md). Cron jobs now run as real Slack thread sessions with full MCP access, and the ignore gate has proper authorization, validation, and user-level tools.

### Features

- **Cron: real Slack thread sessions.** `cron_jobs` now stores `slack_team_id`, `slack_channel_id`, and `slack_thread_ts`. Cron runs resume the same session humans chat in, sharing history and context. [docs/findings/2026-05-25-cron-ignore-audit.md](docs/findings/2026-05-25-cron-ignore-audit.md)
- **AgentManager: `isLive(sessionId)`.** Transports can check whether a session has an active SDK Query loop. Used by CronScheduler to skip a run when humans are actively chatting (humans get priority).
- **MCP tools: `ignore_user` / `unignore_user`.** Managers and approvers can now ignore a user across all threads, not just one thread. Complements the existing `ignore_thread` / `unignore_thread` pair. `src/gateway/slack/mcp-tools.ts`
- **Duration parser: strict validation.** Exported `parseDuration()` rejects invalid suffixes (e.g. `5x`), decimals (`1.5h`), and durations >24h. Shared by MCP tools and slash `/ignore`. `src/gateway/slack/mcp-tools.ts`

### Fixes

- **Cron: silent event drop.** Root cause was synthetic `cron:*` session keys with no registered route. Fixed by passing real Slack keys and registering the route via `onExecute` callback before `sendMessage`. Cron sessions now receive Slack MCP tools and agent events flow normally. `src/gateway/slack/cron-scheduler.ts`, `src/gateway/slack/adapter.ts`
- **Cron: race condition.** `#running` was cleared in `finally` after `sendMessage` resolved, not after the agent finished. Overlapping runs could corrupt the session. Now the set is cleared only on `done` or `error` event (or sendMessage throw). `src/gateway/slack/cron-scheduler.ts`
- **remove_cron_job: prefix matching.** The tool description claimed 8-char prefixes worked but `findById()` only did exact UUID match. Added `findByPrefix()` with collision detection. `src/db/cron-jobs.ts`
- **remove_cron_job: clarify soft-delete.** Response now says "deactivated" and the description explains historical runs remain in the database. `src/gateway/slack/mcp-tools.ts`
- **ignore_thread: missing auth.** Any agent could call `ignore_thread`. Added `isManagerOrApprover` guard to `ignore_thread`, `unignore_thread`, `ignore_user`, and `unignore_user`. `src/gateway/slack/mcp-tools.ts`
- **ignore_thread: duration parsing bugs.** Invalid suffixes (`5x` → 5m), decimals (`1.5h` → 1h), and unlimited durations were silently accepted. All rejected by `parseDuration()`. `src/gateway/slack/mcp-tools.ts`
- **ignore_thread: wrong createdBy.** Hardcoded `"agent"` instead of the actual user ID. Now uses `ctx.userId`. `src/gateway/slack/mcp-tools.ts`
- **unignore_thread: silent no-op.** Returned "removed" even when nothing existed. Now returns "no active ignore for this thread" when the delete count is zero. `src/db/ignores.ts`, `src/gateway/slack/mcp-tools.ts`
- **Schema migration: cron_jobs Slack keys.** Backfills existing rows by copying `channel_id` → `slack_channel_id` and `thread_ts` → `slack_thread_ts` so legacy cron jobs continue to work. `src/db/schema.ts`

### Internal

- Tests rewritten for new cron lifecycle: legacy-key skip, live-session skip, event-driven completion, error propagation, prefix lookup. `tests/cron-scheduler.test.ts`
- `postResult()` dead code removed from `CronScheduler`. `src/gateway/slack/cron-scheduler.ts`
- `SlackContext` gains `teamId` field so MCP-created cron jobs store the workspace ID. `src/gateway/slack/mcp-tools.ts`, `src/gateway/slack/adapter.ts`
