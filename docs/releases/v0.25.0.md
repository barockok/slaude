## v0.25.0 — per-source gather: curated pages survive high-volume sources

A feature release that fixes a retrieval-quality bug where a single curated
page could be drowned out of the candidate set by a high-volume source. Full
writeup: [2026-06-15 per-source-gather](../findings/2026-06-15-per-source-gather.md).

### Features

- **Knowledge: per-source candidate `gather()`.** `kb_search` and `kb_think`
  previously pooled *all* allowed KB sources into one ranked candidate set, so a
  high-volume source — e.g. a bulk auto-generated corpus of thousands of
  near-empty model stubs — could monopolise the gather by sheer count. A small
  curated page never entered the pool, so the reranker never got to sort it up:
  a directory-style lookup resolved fine in a warm-context thread (the page was
  already in context) but failed in a cold thread ("the brain does not contain
  that data"). The new `src/knowledge/gather.ts` fans out one `search` per
  allowed source (single-source scope each), takes the top-K from *each*, then
  merges and re-ranks the union by the engine's own `rerank_score`. Every source
  — including small curated ones — is guaranteed candidate slots, so no source
  can crowd others out by volume; the reranker then sorts the empty stubs down
  and the curated page up. `kb_search` routes through `gather()`, and
  `kb_think`'s zero-citation cross-check uses it too, so an uncited-but-present
  page surfaces in `search_fallback` instead of more bulk-corpus noise.
  (`src/knowledge/gather.ts`, `src/knowledge/mcp-tools.ts`)

### Internal

- **gbrain stays stock — no fork.** The fix only orchestrates the engine's
  existing `search` op differently; it does not patch the dependency. This is the
  proving ground — once validated in production, per-source ranking is intended
  for promotion upstream into gbrain itself.
- **Failure posture is explicit.** A single source failing drops to `[]`; if
  *every* source fails the brain is genuinely down, so `gather()` rethrows and the
  handler maps it to an error rather than papering over a dead brain with empty
  results.
- **Known limitation (follow-up).** `kb_think`'s *primary* gather still runs
  inside the engine's `runThink` and is out of the wrapper's reach, so synthesis
  remains pooled; the per-source cross-check rescues the answer via
  `search_fallback`. Re-pointing the synthesis candidate step at the shared
  `gather()` is deferred as higher-risk.

### Tests

- New `tests/gather.test.ts` — deterministic injected-call units (fan-out,
  flood-survival, dedup-by-slug, slugless-dedup, finalLimit + sort, score
  fallback, single-source fallback, partial-failure tolerated, total-failure
  rethrows) plus one self-contained real-engine smoke. `tests/brain-mcp-tools.test.ts`
  updated for the new `kb_search` contract. Patch coverage 100%.
