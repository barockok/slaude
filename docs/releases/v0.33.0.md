## v0.33.0 — Task visibility, version introspection, per-turn 1on1 awareness

This release adds three observability and session-state improvements — live task progress visible on Slack during long turns, a `version` introspection tool, per-turn 1on1 awareness — plus a cron credential bug that slipped through the v0.32.0 fix.

### Features

- **Gateway: live task visibility on Slack surface.** `TaskCreate` and `TaskUpdate` tool calls are now intercepted and mirrored to Slack in real time. When the agent creates its first task, a live message appears; subsequent status updates edit that message in-place so you see `⏳ in_progress → ✅ completed` without waiting for the turn to end. The message stamps `Tasks ✅` when all tasks are completed or deleted, and state is cleared at the start of each new user turn. See `src/gateway/core/gateway.ts`.

- **Session MCP: `version` tool.** `mcp__slaude_session__version` returns the deployed slaude semver and the bundled `claude-agent-sdk` version, both baked from `package.json` at build time. Useful for the agent to self-report its build when asked, or for operators to verify deployed version without shelling in.

- **Gateway: per-turn 1on1 lock state in channel envelope.** The `<channel>` envelope injected on every inbound turn now includes `one_on_one="true"` and `locked_user="<@Uxxx>"` attributes when a `/1on1` lock is active. Previously the 1on1 session-mode block was injected once at session boot — over a long conversation the model could gradually lose awareness that it was in exclusive mode. The per-turn signal keeps the constraint visible regardless of context compaction.

### Fixes

- **Cron: `oauthUser` not captured when agent creates a cron in /1on1.** The `/cron-add` slash command correctly captured the initiator's identity via `OneOnOne.find()`, but the `add_cron_job` MCP tool handler did not. Cron jobs created by the agent during a 1on1 always fired as the agent's own identity, bypassing both the `CLAUDE_CONFIG_DIR` isolation and the private-MCP credential scoping introduced in v0.32.0. `adminHandlers.addCronJob` now looks up the live lock and passes `oauthUser` to `CronJobs.create`, matching the slash command path. (#68)
