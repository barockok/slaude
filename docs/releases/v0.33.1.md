## v0.33.1 — Task visibility mid-turn fix + completedAt timestamps

Patch for a regression in the v0.33.0 task visibility feature: task updates went dark when the user sent a message while a turn was still running.

### Fixes

- **Gateway: task state preserved across mid-turn inbound messages.** `tasksRef` and `tasksMap` were reset in the inbound handler whenever a new Slack message arrived — even if the previous agent turn was still in flight. Any `TaskUpdate` from the still-running turn then silently dropped at the `tasksMap?.has()` guard, making the live task block go dark. Fix: moved the reset to the `done` event (after the final stamp), where it belongs. New inbound messages no longer wipe in-flight task state.

- **Gateway: completed tasks now show `HH:MM:SS` timestamp.** When a task moves to `completed`, the current wall-clock time is stamped as `completedAt` and rendered inline in the task block as `_(HH:MM:SS)_`. The final "Tasks ✅" stamp at turn-end also carries the timestamps.
