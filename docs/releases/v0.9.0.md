## v0.9.0 — Ignore Gate, Cron Routing, Info-Capture Tools

Major feature release. Agent can now mute threads/users, schedule recurring tasks via cron, and capture richer Slack context through new MCP tools. Test coverage boosted to 97%+ function / 99%+ line.

### Features

- **Slack: Ignore Gate — thread/user muting.** New `/ignore` and `/unignore` slash commands let manager/backup/approvers mute noisy threads or users. Ignored events drop before engagement gates with metric label `ignore_gate`. Agent can also self-ignore a thread via `ignore_thread` MCP tool. Data lives in `ignores` table (user_id + thread_ts + reason + actor + expiry). `IgnoreGate` class evaluated in adapter drop chain after blocklist, before engagement checks.
- **Slack: Cron Routing — scheduled recurring tasks.** Full cron scheduler with expression parser, next-run computation, and per-second polling. New `/cron` slash commands (add/list/remove) wired into adapter with manager approval gate. Cron jobs stored in `cron_jobs` table with mutex-style execution (no overlapping runs). Expression syntax: standard 5-field cron (`0 9 * * 1-5`).
- **Slack: Info-capture MCP tools for richer context.** New MCP tools expose Slack metadata to the agent: `slack_getThreadInfo` (participant list, message count), `slack_getUserInfo` (profile, timezone, status), `slack_getChannelInfo` (topic, purpose, member count). Helps agent tailor replies with thread context instead of guessing.
- **Knowledge: tag-based KB search with KB-first enforcement.** Knowledge base queries now support tag filtering. Tag search integrated into MCP `search_knowledge` tool. KB-first stance strengthened: agent must check KB before acting on unfamiliar domains. Enforcement lives in soul runtime baseline.

### Fixes

- **Agent: remove token-budget Slack thread alerts.** Token-budget threshold alerts previously spammed Slack threads on every turn near limit. Removed — alerts now log-only via `session-mcp.ts` introspection.
- **Tests: remove stale env.test.ts.** Covered functions removed in earlier refactor. Was causing false-positive test failures.
- **CronScheduler: type assertion for mock.calls access.** Fixed `any` cast in test mock access that broke under strict `bun test`.

### Internal

- **Test coverage: 97%+ function, 99%+ line.** Added comprehensive suites for ignore gate (`tests/ignore-gate.test.ts`), cron scheduler (`tests/cron-scheduler.test.ts`), info-capture MCP tools (`tests/slack-mcp-tools.test.ts`), and command handlers (`tests/commands.test.ts`).
- **Findings docs:** `docs/findings/2026-05-24-ignore-gate-and-cron-routing-design.md` (design spec) and `docs/findings/2026-05-24-ignore-gate-and-cron-routing.md` (implementation plan + learnings).
