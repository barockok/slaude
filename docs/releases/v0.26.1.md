## v0.26.1 — approval gate auto-denies instead of hanging the MCP server

A patch release fixing a `stream_closed` cascade caused by approval gates that
waited forever for a human click.

### Fixes

- **Approval: default to a 300s timeout when the soul config is `0`/unset.**
  In-process MCP tool handlers that block too long trip the SDK's internal MCP
  timeout, which breaks not just the blocked call but every subsequent call on
  the same server (observed cascade: `kb_memoize` → `reply`, `kb_search`,
  `list_kbs` all fail with `stream_closed`). The dominant trigger is
  `kb_memoize`'s implicit approval gate: `approvalTimeoutSeconds` defaults to `0`
  (wait forever) in `SoulDataSchema`, and most deployed personas never set it, so
  a gate with no clicker would hang indefinitely. The gate now falls back to 300s
  (`soulData().approvalTimeoutSeconds || 300` in `src/gateway/core/gateway.ts`),
  after which the request cleanly resolves to `approved: false` — no cascade. An
  explicit `approvalTimeoutSeconds` in `SOUL.md` still overrides the default;
  the only behavior removed is unbounded "wait forever" (`0`), which was the bug.
- **Approval: the Slack card now shows when it auto-denies.** The Block Kit
  approval card renders a context line — `⏳ Auto-denies in *300s* if no one
  clicks.` — using the live timeout value, so approvers know the gate is bounded
  (`src/gateway/slack/approval-gate.ts`).
