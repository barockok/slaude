## v0.23.4 — docs: kb_memoize in the how-slaude-works skill

A docs-only patch following v0.23.3's `kb_put_page` → `kb_memoize` rename. The
`how-slaude-works` self-documentation skill described KB *reads* (`kb_search`,
`kb_think`) but never named the durable-write tool, and an operational comment
still referenced the old tool name.

### Changes

- **`how-slaude-works` skill: name `kb_memoize` for durable writes.** The KB-first
  section now tells the agent to record durable knowledge with `kb_memoize`
  (array of up to 20 pages, `[[wikilinks]]`, batch related notes). Previously the
  skill covered only the read path. (`skills/how-slaude-works/SKILL.md`)
- **Comment accuracy.** The brain source-bootstrap comment in the gateway now
  refers to `kb_memoize` instead of the removed `kb_put_page`.
  (`src/gateway/core/gateway.ts`)

No behavior change; no schema or tool change. The runtime baseline prompt already
moved to `kb_memoize` in v0.23.3.
