## v0.13.0 — Session reload for live MCP/plugin pickup

This release adds a `reload_session` MCP tool so managers and approvers can gracefully recycle a live session. Newly installed MCP servers, plugins, or skills are picked up on the next inbound message without restarting the slaude process.

### Features

- **`mcp__slaude_slack__reload_session` tool.** Agent exposes a tool that closes the SDK Query loop cleanly (no "process exited" errors) and marks the session idle. The next user message boots a fresh Query with freshly-resolved MCPs, plugins, and skills. Gated to manager/approver authorization via the existing `isManagerOrApprover` check. `src/gateway/slack/mcp-tools.ts`, `src/agent/manager.ts`
- **Graceful reload with error suppression.** `AgentManager.reload()` sets a `reloading` flag before closing the prompt iterable. If the SDK throws during teardown, the error is suppressed so Slack users never see a scary stack trace. `src/agent/manager.ts`
- **Wired for all route lifecycles.** `reloadSession` callback is bound on initial route creation, route mutation on subsequent turns, and cron-scheduler routes. `src/gateway/slack/adapter.ts`
